## 🎯 Feature Overview

Move from **Passive Execution** (waiting for commands to finish) to **Active Monitoring** (interacting with commands as they run).

### Current Limitations (v2.11.0)
- ❌ Commands execute completely, then return full stdout/stderr
- ❌ No real-time log feedback to user
- ❌ No interactive stdin support (prompts hang)
- ❌ No process cancellation (kill switch)

### Target Capabilities (v2.12.0)
- ✅ Real-time streaming of stdout/stderr to VS Code Output Channel
- ✅ Interactive stdin support for terminal prompts
- ✅ Process management with cancellation tokens
- ✅ Graceful timeout handling (>30s auto-cancel)

## 🏗️ Three Milestones

### Milestone 1: Core Streaming Interface (Day 1-2)
**Goal**: Implement streaming infrastructure via child_process.spawn

**Deliverables**:
- AsyncCommandRunner.ts with spawn() method
- ProcessHandle interface with real-time events
- Timeout logic (SIGTERM → SIGKILL escalation)
- Stream buffering optimized for 10,000+ lines
- Comprehensive streaming tests

**Coverage Gain**: +0.4%

**Key Implementation**:
```typescript
interface ICommandRunner {
  exec(command: string): Promise<{ stdout: string; stderr: string }>;
  spawn(command: string, options?: SpawnOptions): ProcessHandle;
}

interface ProcessHandle {
  onData: (callback: (data: string) => void) => void;
  onError: (callback: (error: Error) => void) => void;
  onExit: (callback: (code: number) => void) => void;
  kill: () => void;
  isRunning: () => boolean;
}
```

**Test Scenarios**:
- [ ] Data arrives and triggers onStdout callback
- [ ] Large buffers (10,000 lines) don't crash
- [ ] Process auto-killed after 30s default timeout
- [ ] SIGTERM sent first, SIGKILL after 5s
- [ ] Timeout is configurable
- [ ] Timeout fires onExit event
