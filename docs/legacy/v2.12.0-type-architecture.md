**Status**: ✅ Locked In (Ready for Implementation)

**Date**: February 27, 2026

## 📋 Overview

The v2.12.0 feature requires a coordinated set of new types that enable:

1. **PlanState** - Executor lifecycle with SUSPENDED_FOR_PERMISSION state
2. **ProcessHandle** - Real-time process interaction with stdin/stdout
3. **SuspendedExecutionState** - Crash-proof state persistence
4. **StreamingIO** - Prompt detection and event handling
5. **ExecutorExtensions** - Implementation guide for Executor class

This document describes the interaction between these types.

---

## 🔄 Type Interaction Diagram

```
┌─────────────────────────────────────────────────────────────────────┐
│                        Executor.executePlan()                       │
│                      (Main execution loop)                          │
└────────────────┬────────────────────────────────────────────────────┘
                 │
                 ├─→ [PlanState.EXECUTING]
                 │   For each step in plan:
                 │     ├─→ executeRun(step) → ProcessHandle
                 │     │   ├─→ handle.onData() → emit streaming output
                 │     │   ├─→ handle.onPrompt() → PROMPT DETECTED!
                 │     │   └─→ handle.onExit() → StepResult
                 │     │
                 │     └─→ Check result.requiresUserInput flag
                 │         │
                 │         ├─ FALSE: Continue to next step
                 │         │
                 │         └─ TRUE: SUSPEND EXECUTION
                 │           ├─→ [PlanState.SUSPENDED_FOR_PERMISSION]
                 │           ├─→ Create SuspendedExecutionState
                 │           ├─→ Capture fileSnapshot (hashes)
                 │           ├─→ Persist to codebaseIndex
                 │           ├─→ Emit executor:suspended event
                 │           └─→ Return control to UI
                 │
                 └─→ [User responds to prompt in OutputChannel]
                     └─→ Extension calls executor.resume(userInput)
                         ├─→ verifyStateIntegrity()
                         │   ├─→ Check all files match hashes
                         │   ├─→ If conflicts: emit integrity_failed & return
                         │   └─→ If valid: continue
                         ├─→ handle.sendInput(userInput)
                         ├─→ [PlanState.EXECUTING]
                         └─→ Main loop continues with remaining steps
```

## 📁 Type Files & Their Responsibilities

### 1. **PlanState.ts** - Executor Lifecycle States

```typescript
enum PlanState {
  IDLE,
  PLANNING,
  EXECUTING,
  SUSPENDED_FOR_PERMISSION,  // ← NEW for v2.12.0
  PAUSED,
  COMPLETED,
  FAILED
}
```

**When to use**:
- Store in: `Executor.executorState`
- Check in: `if (executor.getState() === PlanState.SUSPENDED_FOR_PERMISSION)`
- Set to SUSPENDED when: `result.requiresUserInput === true`
- Set to EXECUTING when: `executor.resume(userInput)` called

**File**: `src/types/PlanState.ts` (28 lines)

### 2. **ProcessHandle.ts** - Process Interaction Interface

```typescript
interface ProcessHandle {
  onData(callback: (data: string) => void): void;
  onError(callback: (error: Error) => void): void;
  onExit(callback: (code: number) => void): void;
  onPrompt(callback: (text: string) => void): void;      // ← NEW
  sendInput(data: string): void;                           // ← NEW
  kill(): void;
  isRunning(): boolean;
  getExitCode(): number | undefined;
  getPID(): number | undefined;
}
```

**When to use**:
- Return from: `commandRunner.spawn(command, options)`
- Call in executeRun: `handle.onData(...)`, `handle.onExit(...)`
- For interactive commands: `handle.onPrompt(...)`, `handle.sendInput(...)`
- For cancellation: `handle.kill()`

**File**: `src/types/ProcessHandle.ts` (128 lines)

### 3. **SuspendedExecutionState.ts** - State Persistence

```typescript
interface SuspendedExecutionState {
  planId: string;
  stepIndex: number;
  step: ExecutionStep;
  timestamp: number;
  remainingSteps: ExecutionStep[];
  context: {
    lastStdout: string;
    lastStderr: string;
    fileSnapshot: Record<string, string>;  // file → hash
    completedSteps: Map<number, StepResult>;
    processExitCode?: number;
    processPID?: number;
  };
  promptText?: string;
  metadata?: { ... };
}
```

**When to use**:
- Create when: Plan needs to suspend (step.requiresUserInput)
- Persist to: `codebaseIndex.setPlanSuspension(planId, state)`
- Retrieve from: `codebaseIndex.getPlanSuspension(planId)` (crash recovery)
- Read from: `executor.getSuspendedState()`

**File**: `src/types/SuspendedExecutionState.ts` (167 lines)

### 4. **StreamingIO.ts** - Prompt Detection & Events

```typescript
type StreamingEvent =
  | { type: 'data'; stepId: number; data: string; isStderr: boolean }
  | { type: 'prompt'; stepId: number; promptText: string }
  | { type: 'suspended'; stepId: number; planId: string }
  | { type: 'process_exit'; stepId: number; exitCode: number }
  | ...;

interface PromptPattern {
  pattern: RegExp;
  name: string;
  suggestedInputs?: string[];
  defaultInput?: string;
}

const DEFAULT_PROMPT_PATTERNS = [
  { pattern: /npm.*\(y\/n\)/i, name: 'npm_proceed', ... },
  { pattern: /\[Y\/n\]/, name: 'generic_yes_no', ... },
  // ... more patterns
];

function detectPrompt(text: string): PromptPattern | null;
function looksLikePrompt(text: string): boolean;
```

**When to use**:
- Import patterns: `import { DEFAULT_PROMPT_PATTERNS, detectPrompt }`
- Check each data chunk: `if (detectPrompt(stdout)) { ... }`
- Get suggestions: `getSuggestedInputs(stdout)`
- Emit events: `onStreamData(stepId, data, isStderr)`

**File**: `src/types/StreamingIO.ts` (174 lines)

## 🔗 Type Dependency Graph

```
PlanState.ts
  └─→ Used by: Executor.executorState

ProcessHandle.ts
  ├─→ Created by: CommandRunner.spawn()
  ├─→ Used in: executeRun(step)
  └─→ Extends with: onPrompt(), sendInput()

StreamingIO.ts
  ├─→ Provides: DEFAULT_PROMPT_PATTERNS
  ├─→ Provides: detectPrompt(), looksLikePrompt()
  └─→ Used in: executeRun() for prompt detection

SuspendedExecutionState.ts
  ├─→ Extends: ExecutionStep, StepResult
  ├─→ Created in: executePlan() when requiresUserInput
  ├─→ Stored in: codebaseIndex.planSuspensions[planId]
  ├─→ Retrieved in: crash recovery flow
  └─→ Used by: verifyStateIntegrity()

ExecutorExtensions.ts (documentation)
  └─→ Shows how all above types fit into Executor class
```
